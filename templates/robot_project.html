Death-Stranding: Autonomous Delivery Robot

Abstract

The project involves constructing an autonomous robot that can navigate a map, pick up packages from houses, and deliver them to the correct locations. The robot must handle three types of objects: a cylinder, a rectangular prism, and a dumbbell. The map features six locations, with two designated for each type of object. The robot uses an Arduino Nano Every for control, DC motors, infrared sensors for line following, and servomotors for object manipulation.

Introduction

The project's goal is to develop a mailman robot that sorts and delivers objects to the correct houses. This has potential applications in large-scale enterprises like Amazon, where autonomous robots can streamline the sorting and delivery process.

Map and object placement

Project Strategy

The group focused on three main areas:

  • Computer Modeling: Designing a prototype using Fusion 360 and testing algorithms on Simulink.
  • Component Selection and Construction: Choosing appropriate components and materials for the robot's construction.
  • Testing and Model Validation: Testing the prototype and refining the model based on the results.

Modeling

The robot uses a PID controller for line following. Simulink was used to model the robot's behavior and test the PID coefficients. The model includes simulations of sensors and motors to ensure the robot can follow the line accurately.

Realization in Arduino

The Arduino code includes functions for calibrating sensors, implementing the PID controller, detecting branches in the line, and scanning for objects. The robot uses a combination of infrared and ultrasonic sensors to navigate and recognize objects.

Flowchart of the piloting code

Program

The robot's algorithms are written in Arduino C++. Key functions include:

  • Sorting Algorithm: Determines the order in which the robot should visit houses to sort the objects.
  • Line Following Algorithm: Uses infrared sensors and a PID controller to keep the robot on the line.

Project Management

The project was managed using tools like Gantt charts, GitLab, and status reports. The group faced challenges such as component availability, design issues, and time management. A SWOT analysis was conducted to identify strengths, weaknesses, opportunities, and threats.

Conclusion

The project involved significant learning and problem-solving. Key lessons included the importance of accurate modeling, effective piloting, gripper design, and overall robot design. The group successfully developed a prototype that could navigate and sort objects, although there were challenges and areas for improvement.

This project taught valuable lessons about system design, component selection, and the importance of iterative testing and refinement as well as key skills in project management and teamwork.
